# Lightory e2e tests

Playwright end-to-end tests for the browser-only Lightory runtime.

## Scope

The suite launches the built standalone CLI, opens the browser UI, and verifies
that hook events flow through the local server into the canvas and WebSocket
message stream.

Covered today:

- `@area:standalone` hook-driven lifecycle from `SessionStart` through
  `SessionEnd`
- browser WebSocket transport
- canvas agent overlay visibility
- settings updates through the browser UI

Not covered here:

- real OpenCode or Codex CLI execution
- browser compatibility outside Chromium
- visual polish beyond DOM/test-hook assertions
- desktop shell, mobile, or pad shells

## Running

```bash
npm run build
npm run e2e

npm run e2e -- --headed
npm run e2e -- --grep "@area:standalone"
npm run test:report
```

The fixture starts `dist/cli.js` on a free localhost port and stores temporary
state under the OS temp directory. It does not require VS Code, Electron, or an
installed extension host.

## Adding Tests

- Place browser-only tests under `e2e/tests/standalone/`.
- Use the fixture from `e2e/fixtures/standalone.ts`.
- Drive server hook behavior through `e2e/helpers/hooks.ts`.
- Assert user-visible browser behavior first; use recorded WebSocket messages
  only for protocol details that are not visible in the UI.
