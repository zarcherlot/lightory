## 当前开发状态

更新时间：2026-07-08

当前工程已经从开发者观察工具转向 Lightory agent 教育软件 MVP。现阶段已实现：

- 儿童友好角色：天气预报员、穿衣建议员、出行提醒员、小队队长。
- 编辑模式 / Play 模式。
- 角色头顶能力卡。
- 能力卡拖拽建立连接。
- 连接后的半透明拱形曲线。
- 任务卡与 `运行小队` 按钮。

- 上游失败后的重试与停止下游执行。

状态说明：

- `已完成`：当前代码已实现主要行为。
- `部分完成`：已有 MVP，但仍缺关键体验或产品细节。
- `未完成`：当前尚未实现。
- `调整`：当前实现与原需求有意不同，通常是为了验证效果。

## 产品目标

把 Lightory 打造成适合小朋友学习 agent 设计与编排的教育软件。

核心学习目标：

- 理解一个 agent 是一个有能力的角色。
- 理解 agent 之间可以传递信息。
- 理解复杂任务可以拆成多个角色协作完成。
- 通过可视化运行过程观察输入、处理、输出、传递。
- 用低门槛交互替代复杂 workflow、DAG、console 操作。

## 核心场景（部分完成）

第一条落地场景：

> 明天去学校 / 公园，需要准备什么？

基础角色：

- [已完成] 天气预报员：查询天气，输出天气卡。
- [已完成] 穿衣建议员：根据天气卡输出穿衣卡。
- [已完成] 出行提醒员：根据天气卡输出出行卡。
- [已完成] 小队队长：收集其他卡片，输出最终准备清单。

目标链路：

```text
天气预报员 -> 穿衣建议员 -> 小队队长
天气预报员 -> 出行提醒员 -> 小队队长
```

当前状态：

- [已完成] 支持通过能力卡拖拽建立上述依赖关系。
- [已完成] Play 模式按依赖顺序执行：天气先执行，穿衣/出行等待天气完成，队长等待穿衣/出行完成。
- [部分完成] 结果目前只在 Console 展示；Play 模式不在主界面绘制结果卡或卡片飞行动画。

## P0：编辑模式与 Play 模式（已完成）

必须先建立两个清晰模式。

### 编辑模式

用途：让小朋友设计 agent 小队。

表现：

- [已完成] 角色头顶显示“能力卡”。
- [已完成] 能力卡表示这个角色可以输出什么。
- [已完成] 小朋友可以拖拽能力卡到另一个角色身上，建立协作关系。
- [已完成] 成功后显示一条半透明拱形曲线连接两个角色。
- [调整] 当前版本为了观察效果，曲线常驻显示；后续可恢复为短暂反馈或按交互显示。

示例：

```text
天气预报员头顶：[天气卡]
穿衣建议员头顶：[穿衣卡]
小队队长头顶：[计划卡]
```

### Play 模式

用途：运行任务，观察 agent 工作过程。

表现：

- [已完成] 隐藏编辑模式下的头顶能力卡。
- [已完成] 恢复角色运行状态表现。
- [已完成] 天气预报员继续显示天气图标轮播。
- [已完成] 其他角色显示独立运行图标槽位：穿衣、出行、队长。
- [调整] 运行过程中真实结果只显示在 Console，主界面不显示结果卡。
- [调整] 运行模式不显示连接线和卡片传递动画。

规则：

> 编辑模式显示“我能给别人什么”；Play 模式显示“我正在做什么、我产出了什么”。

## P1：能力卡拖拽建立连接（已完成）

这是替代“手动画线”的核心交互。

需求：

- [已完成] 每个角色在编辑模式下头顶显示一张或多张能力卡。
- [已完成] 能力卡可被拖拽。
- [已完成] 拖拽时，可接收该卡片的角色高亮。
- [已完成] 拖到可接收角色身上后建立连接。
- [部分完成] 拖到不可接收角色身上时显示失败提示；卡片依赖浏览器默认拖拽回弹。
- [已完成] 连接成功后显示反馈。

成功反馈：

- [已完成] 目标角色高亮。
- [部分完成] 能力卡吸附一下后回到原位置。
- [已完成] 两个角色之间出现半透明拱形曲线。
- [已完成] 曲线上显示卡片类型，例如 `天气卡`。
- [调整] 当前版本曲线常驻显示，用于观察连线效果；淡出版本后续再切回。

失败反馈：

- [已完成] 支持类似“这个角色不需要这张卡，可以交给某某角色”的提示。

## P2：半透明拱形曲线连接反馈（已完成）

连接线原设计是不常驻，只作为短暂反馈和回放提示。

需求：

- [已完成] 连接成功时出现。
- [已完成] 曲线为拱形，连接两个角色。
- [已完成] 带箭头方向。
- [已完成] 有透明度，避免干扰画面。
- [已完成] 颜色跟卡片类型一致。
- [调整] 当前版本不淡出，常驻显示。
- [调整] 当前版本平时不隐藏，作为可视化编排结果。

额外交互：

- [已完成] 鼠标悬停某个角色时，可高亮该角色相关连接，并弱化无关连接。
- [已完成] 点击角色时，可持续显示它的上游和下游连接；再次点击取消聚焦。
- [已完成] Play 时连接曲线保持显示；每批下游角色运行前，会重新高亮对应信息流连接。

## P3：任务卡片与 Play Button（已完成）

任务执行需要明确的 Play button。

需求：

- [已完成] 任务卡片上显示任务目标。
- [已完成] 有明确的 `运行小队` 按钮。
- [已完成] 点击后进入 Play 模式。
- [已完成] Play 模式中禁止编辑能力卡连接。
- [已完成] 可手动回到编辑模式。
- [已完成] 运行结束后自动进入完成态。

状态设计：

```text
[已完成] 未开始：运行小队
[已完成] 运行中：暂停 / 停止
[已完成] 完成：再跑一次（查看回放归入 P7）
[已完成] 出错：修一修 / 再试一次
```

原则：

> 拖角色、拖能力卡是搭建机器；按 Play 是让机器跑起来。

## P4：角色配置二级窗口（部分完成）

不要从 Console 输入天气城市。点击角色后弹出角色配置窗口。

以天气预报员为例：

```text
天气预报员

我负责：查询天气

城市：
[ 上海 ]

日期：
[ 明天 v ]

我会输出：
[x] 温度
[x] 是否下雨
[x] 风力
[ ] 空气质量

[保存] [运行这个角色]
```

需求：

- [已完成] 点击角色打开二级窗口。
- [已完成] 支持编辑角色参数；当前天气、穿衣、出行、队长和扩展儿童角色都有简单配置入口。
- [已完成] 支持保存配置。
- [已完成] 保存后角色卡片或面板显示当前配置。
- [已完成] 天气预报员不再从 Console 读城市，运行时使用界面配置生成的任务 Markdown。

进阶需求：

- [已完成] 提供 `简单模式 / Markdown 模式`。
- [已完成] 简单模式用表单；天气、穿衣、出行、队长和扩展角色均有表单。
- [已完成] Markdown 模式允许编辑底层任务说明。
- [部分完成] 表单和 Markdown 尽量双向同步；表单改动会生成 Markdown，切回简单模式时会尽量解析当前 Markdown，但只覆盖当前模板中的关键字段。

原则：

> Markdown 可以存在，但不应该是儿童用户的唯一入口。

## P5：能力卡与结果卡区分（部分完成）

需要明确区分两类卡片，避免交互混乱。

### 能力卡

出现于编辑模式。

表示：

```text
这个角色能产出什么
```

例子：

- [已完成] 天气卡。
- [已完成] 穿衣卡。
- [已完成] 出行卡。
- [已完成] 计划卡。
- [未完成] 检查卡。

用途：

- [已完成] 拖给其他角色。
- [已完成] 建立协作关系。

### 结果卡

出现于 Play 模式。

表示：

```text
这个角色实际产出了什么
```

用途：

- [部分完成] 展示真实运行结果；当前只在 Console 展示完整输出。
- [调整] 沿已建立连接传递给下一个角色的可视化动画已取消，避免运行模式画面干扰。

## P6：信息流动画（部分完成）

Play 模式下，运行过程要可观察。

需求：

- [已完成] 角色执行时显示对应状态动画。
- [已完成] 天气预报员保留天气图标轮播。
- [已完成] 穿衣、出行、队长拥有独立运行图标槽位。
- [部分完成] 角色完成后生成结果；当前只在 Console 展示。
- [调整] Play 模式不显示结果卡沿连接关系传递的动画。
- [已完成] 接收角色在依赖完成后进入运行状态。
- [部分完成] 最终由小队队长生成结果清单；依赖角色 markdown 任务执行结果。

天气流程示例：

```text
天气预报员生成：天气结果卡
天气结果卡飞向：穿衣建议员
穿衣建议员生成：穿衣建议卡
穿衣建议卡飞向：小队队长
小队队长生成：最终准备清单
```

## P7：任务回放（未完成）

适合教学使用。

需求：

- [未完成] 任务完成后提供“查看回放”。
- [未完成] 回放时逐步展示每个角色的输入、处理、输出、传递。
- [未完成] 可以暂停、继续、重播。
- [未完成] 回放时可临时显示连接曲线和卡片流动。

回放信息：

```text

天气预报员输出：天气卡

穿衣建议员输出：穿衣卡
小队队长收到：穿衣卡、出行卡
小队队长输出：准备清单
```

## P8：错误与推荐机制（部分完成）

错误应该转化为学习提示。

需求：

- [已完成] 如果角色缺少输入，角色用气泡提示。
- [已完成] 如果能力卡拖给了不合适的角色，系统推荐合适角色。
- [已完成] 如果任务无法运行，阻止下游继续执行，并对上游失败进行重试；缺少输入卡时会在角色旁显示儿童化提示。
- [部分完成] 不显示部分技术错误给儿童用户；Console 仍可能显示 provider 原始输出。

示例：

```text
我还不知道明天的天气。可以请天气预报员先帮我查一下吗？
```

```text
穿衣建议员需要天气卡，请把天气卡交给它。
```

```text
小队队长还没有收到任何建议卡。
```
