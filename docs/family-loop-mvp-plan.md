# Lightory 家庭闭环 MVP 开发文档

## 目标

把当前 agent 教育软件推进成一个可在 Android 手机和平板上使用的 Web MVP。核心不再是让学生反复编辑复杂场景，而是让学生配置有限小队，围绕高频刚需任务持续产出，家长反馈和任务产出共同形成养成闭环。

## 设备与 Web 架构

### 目标设备

- Android 手机：360px 到 480px 宽度，纵向为主。
- Android 平板：600px 到 1280px 宽度，横向和纵向都要可用。
- 桌面浏览器：保留现有开发调试能力，但不作为 MVP 首屏体验的唯一标准。

### 响应式布局原则

- 手机：任务流优先，console/对话放底部抽屉，角色小队用横向滑动栏，画布退为背景和状态展示。
- 平板：左侧任务/卡片，中间场景，右侧对话或家长闭环面板。
- 桌面：保留现有画布 + 右侧 console 布局。

### 前端分层

- `App`：运行状态、全局面板编排、运输层接入。
- `RoleDock`：有限角色选择与小队槽位入口。
- `EducationModeOverlay`：卡片连接、运行状态、结果卡展示。
- `RoleTaskConsole`：交互式对话入口，承载学生输入、系统追问、任务确认。
- 后续新增 `ScenarioPanel`：暑假新闻 / 每日 TODO 两类任务的入口、进度和提醒状态。
- 后续新增 `ParentLoopPanel`：家长已读、点赞、补充、采纳的回流展示。

### 后端分层

- WebSocket 协议继续作为浏览器与本地服务的实时通道。
- 角色任务继续复用 `startRoleTask` / `roleTaskRunner`。
- Console 先走轻量规则对话，后续再接入角色任务或 LLM。
- 提醒 MVP 先实现本地状态与 UI 计划，不直接依赖系统推送；Android Web 推送和 PWA 放第二阶段。

## 核心闭环

```text
学生选择或配置小队
-> 小队围绕场景产出卡片
-> 学生确认、选择或完成任务
-> 产出可推送给家长
-> 家长已读 / 点赞 / 补充 / 采纳
-> 角色经验、小队属性、流程实用度增长
-> 第二天或下一次任务自动复用
```

成长值来源分三类：

- 工作流产出：准时产出、完成率、复用率、减少遗漏、连续运行。
- 学生互动：选择新闻、完成 TODO、回答追问、修正计划。
- 家长反馈：已读、点赞、补充、采纳、设为提醒。

## 场景一：暑假每日新闻摘抄

### 用户故事

学生暑假每天要摘抄一条新闻。系统每天收集热点新闻，过滤不适合小学生摘抄的内容，让学生在少量候选里选择一条，并定时提醒完成摘抄。

### 角色与卡片

- 新闻收集员：输出 `热点新闻卡`。
- 新闻过滤员：输出 `适合摘抄卡`，过滤暴力、灾害细节、成人政治冲突、低俗娱乐等内容。
- 摘抄推荐员：输出 `候选新闻卡`，给 3 条以内可选新闻。

本阶段不再为“交互选择、提醒、总结”额外拆角色：

- 交互选择由 console 内 TUI 完成。
- 摘抄总结由 console 针对勾选新闻生成。
- 定时提醒放到后续通用提醒能力，不进入新闻摘抄首轮角色池。

### MVP 流程

```text
每日触发 / 手动开始
-> 新闻收集员生成 5-8 条热点
-> 新闻过滤员筛掉不适合内容
-> 摘抄推荐员给 3 个候选
-> console 内 TUI 展示过滤后的候选新闻
-> 学生勾选要摘抄的新闻
-> console 生成适合手抄长度的摘抄总结
```

### 第一阶段边界

- 热点新闻可以先通过用户输入或静态 mock 数据模拟，避免一开始接入新闻源。
- 过滤规则先明确写入角色 prompt，并在 UI 上显示“已过滤原因”。
- 定时提醒暂不进入本轮新闻摘抄 MVP，先把“过滤、选择、总结”闭环做顺。

## 场景二：学生每日 TODO

### 用户故事

学生每天维护自己的 TODO。可以通过对话新增任务、标记完成，系统根据完成进度给个性化回复，并在合适时间提醒。

### 角色与卡片

- 任务记录员：输出 `TODO 卡`。
- 计划拆分员：把大任务拆成可执行小任务，输出 `步骤卡`。
- 进度提醒员：输出 `进度提示卡`。
- 鼓励反馈员：根据完成情况给个性化回复，输出 `鼓励卡`。
- 检查员：发现过多、过难、时间冲突的任务，输出 `检查卡`。
- 家长同步员：生成可给家长看的简短进度摘要。

### MVP 流程

```text
学生输入：今天要做数学、阅读、练字
-> 任务记录员生成 TODO
-> 计划拆分员整理顺序
-> 学生输入：完成阅读
-> 进度提醒员更新进度
-> 鼓励反馈员给个性化回复
-> 家长同步员生成今日进度摘要
```

### 第一阶段边界

- TODO 数据先存在前端运行态，刷新丢失可接受；第二阶段持久化。
- Console 支持新增和完成两类自然语言意图。
- 提醒先做 UI 状态，不做系统级通知。

## MVP 迭代顺序

1. 移动/平板响应式骨架：console 底部化、角色栏适配、画布不遮挡主要操作。
2. Console 对话闭环：输入后返回分类、下一步建议、可执行确认。
3. 新闻摘抄场景 mock 闭环：候选新闻、过滤说明、交互勾选、摘抄文本。
4. TODO 场景轻量闭环：新增、完成、进度提示、鼓励回复。
5. 家长反馈模拟面板：已读、点赞、补充、采纳映射到成长值。
6. 持久化：TODO、新闻选择、成长值、提醒时间写入本地存储或后端状态。
7. PWA / Android 推送：安装到桌面、通知权限、定时提醒。

## 当前首要任务

- 敲定 Web 响应式架构并让 Android 手机和平板可操作。
- 把 console 从回显分类升级成最小对话闭环。
- 为两个典型场景建立角色、卡片和状态模型。
- 每一步完成后跑构建，避免产品探索破坏已有角色运行链路。
